# 第一步：本地代码初始化成 Git，并关联Fork 仓库
进入本地代码文件夹，右键 `Git Bash`

```bash
# 1. 初始化为本地git仓库
git init

# 2. 关联你自己 fork 的远程仓库
git remote add origin https://github.com/你的ID/xxx.git

# 3. 拉一下远端现有代码（保证版本对齐）
git pull origin main --allow-unrelated-histories
```

> 分支名如果是 `master` 就把 main 换成 master
>

# 第二步：把你的本地更新提交
```bash
git add .
git commit -m "更新：xxxx"
```

# 第三步：推送到自己的 Fork 仓库
```bash
git push origin main
```

# 第四步：在 GitHub 网页端发起 PR
1. 打开 Fork 仓库页面
2. 点击 `Compare & pull request`
3. 目标分支选：**原仓库 / 对应分支**
4. 创建 PR，审核合并即可

---

# 额外：以后想同步仓库的最新代码
可以再加一个「上游源」：

```bash
git remote add upstream https://github.com/队友ID/xxx.git
```

以后拉最新代码：

```bash
git pull upstream main
```

---

**如果要提交就必须用git clone下来的代码，不然会显示无相同历史！！！**
