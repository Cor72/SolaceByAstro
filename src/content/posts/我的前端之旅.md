# Template
## 根容器
```html
<div class="app-container" @mousemove="onMouseMove" @mouseleave="onMouseLeave">
```

**整个应用的根容器**，占满全屏。

| 部分 | 含义 |
| --- | --- |
| `class="app-container"` | CSS 类名，控制全屏布局 |
| `@mousemove="onMouseMove"` | **鼠标移动时**触发 `onMouseMove` 函数（显示控制按钮） |
| `@mouseleave="onMouseLeave"` | **鼠标离开容器时**触发 `onMouseLeave`（隐藏控制按钮） |


> `@` 是 Vue 的**事件绑定语法糖**，等价于 `v-on:`。比如 `@click` = `v-on:click`
>

## 域名弃用提示条
```html
<div v-if="isDeprecatedDomain" class="deprecation-banner">
```

**条件渲染**：只有当 `isDeprecatedDomain` 为 `true` 时才显示这个提示条。

```html
<span>当前域名即将弃用，请尽快同步学习数据并使用 </span>
<a href="https://study.mikumod.com" class="new-domain-link">study.mikumod.com</a>
<button class="dismiss-btn" @click="isDeprecatedDomain = false">×</button>

```

| 部分 | 作用 |
| --- | --- |
| `v-if` | Vue 条件指令，`true` 渲染，`false` 移除 DOM |
| `@click="isDeprecatedDomain = false"` | 点击 × 按钮，关闭提示条（直接把变量设为 false） |


## 背景视频（核心视觉）
```html
<transition name="fade" mode="out-in">
```

**Vue 内置过渡组件**，包裹需要动画效果的元素。

| 属性 | 含义 |
| --- | --- |
| `name="fade"` | 使用名为 `fade` 的 CSS 过渡动画（淡入淡出） |
| `mode="out-in"` | **先让旧元素淡出，再让新元素淡入**，避免两个元素同时存在 |


```html
<video 
  :key="currentVideo"
  ref="videoRef" 
  :class="['video-background', { 'video-background-focus-left-mobile': currentVideoName === '1.mp4' }]" 
  :src="currentVideo" 
  autoplay 
  muted 
  loop
  playsinline
  webkit-playsinline
  preload="auto"
  @loadeddata="onVideoLoaded"
  @error="onVideoError"
  @stalled="onVideoStalled"
></video>

```

**背景视频元素**，逐行拆解：

| 属性 | 含义 |
| --- | --- |
| `:key="currentVideo"` | **绑定 key**，当 `currentVideo` 变化时，Vue 会重新创建 video 元素（配合 transition 实现切换动画） |
| `ref="videoRef"` | **模板引用**，在 `<script>` 里可以通过 `videoRef.value` 直接操作这个 DOM 元素 |
| `:class="[...]"` | **动态类名绑定**：基础类 `video-background` + 条件类（如果是 `1.mp4` 且移动端，添加特殊定位类） |
| `:src="currentVideo"` | **动态绑定视频地址**，切换视频时这里会变 |
| `autoplay` | 自动播放 |
| `muted` | **静音播放**（浏览器策略要求自动播放必须静音） |
| `loop` | 循环播放 |
| `playsinline` | iOS Safari：视频**内联播放**，不全屏 |
| `webkit-playsinline` | 旧版 iOS 兼容 |
| `preload="auto"` | 浏览器尽可能预加载视频 |
| `@loadeddata` | 视频数据加载完成时触发 |
| `@error` | 加载失败时触发（自动切换下一个视频） |
| `@stalled` | 视频卡住时触发 |


> `:` 是 `v-bind:` 的简写，用于**动态绑定**属性。没有 `:` 就是静态字符串。
>

```html
<button class="switch-video-btn" @click="switchVideo" :class="{ hidden: !showControls }" 
  @mouseenter="onUIMouseEnter" @mouseleave="onUIMouseLeave" 
  @touchstart="onUITouchStart" @touchend="onUITouchEnd">
```

| 事件 | 作用 |
| --- | --- |

## APlayer 设置持久化
```javascript
const APLAYER_SETTINGS_KEY = 'aplayer_settings'
```

**localStorage 的键名常量**。

```javascript
const defaultAPlayerSettings = {
  volume: 0.7,
  loop: 'all',      // 循环模式：全部循环
  order: 'list',    // 播放顺序：列表顺序
  listFolded: false // 播放列表默认展开
}
```

**默认设置**，用户首次访问时使用。

```javascript
const getAPlayerSettings = () => {
  try {
    const saved = localStorage.getItem(APLAYER_SETTINGS_KEY)
    if (saved) {
      return { ...defaultAPlayerSettings, ...JSON.parse(saved) }
    }
  } catch (e) {
    console.error('读取 APlayer 设置失败:', e)
  }
  return defaultAPlayerSettings
}
```

**读取设置**：

+ 从 `localStorage` 取字符串
+ `JSON.parse` 转对象
+ `{ ...default, ...saved }`：**对象展开合并**，`saved` 里有的覆盖默认，没有的用默认
+ `try/catch`：防止 `localStorage` 被禁用或数据损坏导致崩溃

```javascript
const saveAPlayerSettings = (settings) => {
  try {
    localStorage.setItem(APLAYER_SETTINGS_KEY, JSON.stringify(settings))
  } catch (e) {
    console.error('保存 APlayer 设置失败:', e)
  }
}
```

**保存设置**：对象转 JSON 字符串存入 `localStorage`。

## 控制栏自动隐藏逻辑
```javascript
const toggleFullscreen = async () => {
  await useFullscreenToggle()
}
```

**包装函数**：`useFullscreenToggle()` 返回 Promise（全屏 API 是异步的），用 `async/await` 等待完成。

```javascript
const showControls = ref(true)
```

**控制栏显示状态**：`true` 显示，`false` 隐藏。

```javascript
const inactivityTimer = ref(null)
```

**定时器引用**：存储 `setTimeout` 的 ID，方便清除。

```javascript
const startHideTimer = () => {
  if (inactivityTimer.value) {
    clearTimeout(inactivityTimer.value)  // 清除之前的定时器，防止叠加
  }
  inactivityTimer.value = setTimeout(() => {
    if (!isHoveringUI.value) {  // 如果鼠标不在 UI 上
      showControls.value = false
      document.body.style.cursor = 'none'  // 隐藏鼠标指针
    }
  }, 3000)  // 3 秒后隐藏
}
```

**核心逻辑**：鼠标静止 3 秒后，如果不在操作 UI，就隐藏控制栏 + 鼠标。

```javascript
const onMouseMove = () => {
  showControls.value = true
  document.body.style.cursor = 'default'
  startHideTimer()  // 重新计时
}
```

**鼠标移动时**：显示控制栏，恢复鼠标，重新开始 3 秒倒计时。

```javascript
const onMouseLeave = () => {
  if (!isHoveringUI.value) {
    showControls.value = false
    document.body.style.cursor = 'none'
  }
}
```

**鼠标离开整个应用区域**：立即隐藏（不需要等 3 秒）。

## 视频管理
```javascript
const videoRef = ref(null)
```

**模板引用**：对应模板里的 `ref="videoRef"`，挂载后可以通过 `videoRef.value` 访问 `<video>` DOM 元素。

```javascript
const isR2Domain = window.location.hostname === 'study.mikugame.icu'
const R2_BASE_URL = 'https://studycdn.mikugame.icu/mp4'
```

**域名判断**：旧域名用 Cloudflare R2 CDN 加载视频，新域名用本地路径。

```javascript
const videos = isR2Domain
  ? [ `${R2_BASE_URL}/1.mp4`, ... ]
  : [ '/1.mp4', ... ]
```

**三元表达式**：根据域名选择视频地址数组。

```javascript
const savedVideoIndex = getVideoIndex()
const currentVideoIndex = ref(savedVideoIndex < videos.length ? savedVideoIndex : 0)
const currentVideo = ref(videos[currentVideoIndex.value])
```

**恢复上次选择的视频**：从 `localStorage` 读取索引，越界则回退到 0。

```javascript
const currentVideoName = computed(() => 
  currentVideo.value.split('/').pop()?.split('?')[0] || ''
)
```

**计算属性**：从 URL 提取文件名（如 `1.mp4`）。

+ `split('/')` → 按 `/` 分割
+ `.pop()` → 取最后一段
+ `?.split('?')[0]` → 去掉 URL 参数（如 `?v=123`）
+ `|| ''` → 兜底空字符串

```javascript
const switchVideo = () => {
  currentVideoIndex.value = (currentVideoIndex.value + 1) % videos.length
  currentVideo.value = videos[currentVideoIndex.value]
  saveVideoIndex(currentVideoIndex.value)
}
```

**切换视频**：

+ `% videos.length`：**取模运算**，循环切换（0→1→2→0）
+ 保存新索引到 `localStorage`

## 视频错误处理
```javascript
const aplayer = ref(null)
const aplayerInitialized = ref(false)
const { songs, loadSongs, loading } = useMusic()
const { setupCrossfade, cleanup: cleanupCrossfade } = useCrossfade()
```

**播放器相关状态**：

+ `aplayer`：APlayer 实例
+ `songs, loadSongs`：从 `useMusic` 获取歌曲列表和加载函数
+ `setupCrossfade`：切歌时的淡入淡出效果

```javascript
let videoStalledTimer = null

const onVideoLoaded = () => {
  if (videoStalledTimer) { clearTimeout(videoStalledTimer); videoStalledTimer = null }
  const video = videoRef.value
  if (video) {
    video.play().catch(() => {})  // 尝试播放，失败静默处理
    videoStalledTimer = setTimeout(() => {
      if (video && video.currentTime < 0.5) {  // 3 秒后还没播到 0.5 秒
        console.warn('视频播放异常，自动切换')
        switchVideo()
      }
    }, 3000)
  }
}
```

**视频加载完成后的检测**：

+ 尝试自动播放（浏览器可能阻止，`.catch(() => {})` 静默失败）
+ 设 3 秒定时器：如果视频进度 `< 0.5s`，认为加载失败，自动切换下一个

```javascript
const onVideoError = () => {
  console.error('视频加载失败，切换到下一个')
  switchVideo()
}

const onVideoStalled = () => {
  const video = videoRef.value
  if (video && video.currentTime < 0.5) {
    console.warn('视频卡住，尝试切换')
    setTimeout(() => {
      if (video && video.currentTime < 0.5) switchVideo()
    }, 5000)
  }
}
```

**错误/卡住处理**：直接切换或等待 5 秒后确认再切换。

## 51La 网站统计
```javascript
const load51LaAnalytics = () => {
```

**定义加载统计代码的函数**。51La 是国内的一个网站流量统计服务，类似 Google Analytics。

```javascript
  if (document.getElementById('LA_COLLECT')) return
```

**防止重复加载**。如果页面已经有 `id="LA_COLLECT"` 的脚本，直接返回，避免重复初始化。

```javascript
  const script = document.createElement('script')
  script.charset = 'UTF-8'
  script.id = 'LA_COLLECT'
  script.src = '//sdk.51.la/js-sdk-pro.min.js'
```

**动态创建 script 标签**加载 51La 的 SDK：

+ `charset = 'UTF-8'`：确保中文不乱码
+ `id = 'LA_COLLECT'`：用于上面的重复检测
+ `src` 以 `//` 开头：**协议相对 URL**，自动匹配当前页面的 http/https

```javascript
  script.onload = () => {
    if (window.LA) {
      window.LA.init({
        id: "3OFbo4ER8B3QNm7x",
        ck: "3OFbo4ER8B3QNm7x",
        autoTrack: true,
        hashMode: true
      })
    }
  }
```

**脚本加载完成后初始化**：

+ `window.LA`：51La SDK 挂载到全局的对象
+ `id` / `ck`：站点的唯一标识（这是作者自己的，你复刻时要换成自己的或删掉）
+ `autoTrack: true`：自动追踪页面访问、点击等
+ `hashMode: true`：SPA 路由用 hash 模式（`#/xxx`），需要特殊处理

```javascript
  document.head.appendChild(script)
}
```

**把脚本插入到 **`<head>`** 末尾**，开始加载。

## onMounted —— 组件挂载后的初始化

## initAPlayer —— 初始化播放器

**先加载歌曲列表**。`loadSongs()` 是 `useMusic()` 提供的，可能从本地 JSON 或 API 获取。

**读取用户上次的选择**：

+ `savedMusicIndex`：上次播放到第几首歌

```javascript
    aplayer.value = new APlayer({
      container: document.getElementById('aplayer'),
      fixed: true,
      autoplay: true,
      audio: songs.value,
      lrcType: 3,
      theme: '#2980b9',
```

**创建 APlayer 实例**，逐行解释配置：

| 配置项 | 含义 |
| --- | --- |
| `container` | 挂载到哪个 DOM 元素（前面模板里的 `<div id="aplayer">`） |
| `fixed: true` | **固定定位**，播放器吸附在页面底部，不随滚动移动 |
| `autoplay: true` | 自动播放 |
| `audio: songs.value` | 歌曲列表（响应式引用，要用 `.value`） |
| `lrcType: 3` | 歌词格式：**3 表示 LRC 歌词文件**（时间戳格式 `[00:12.34]歌词`） |
| `theme: '#2980b9'` | 主题色（蓝色） |
| `loop` / `order` | 循环模式、播放顺序（从本地存储恢复） |
| `preload: 'auto'` | 预加载音频 |
| `volume` | 初始音量（从本地存储恢复） |
| `mutex: false` | **不互斥**：允许同时播放多个音频实例 |
| `listFolded` | 播放列表默认折叠状态 |
| `listMaxHeight: '200px'` | 播放列表最大高度 |
| `width: '300px'` | 播放器宽度 |


```javascript

```

**恢复上次播放的歌曲**：如果索引有效，切换到那首歌。

**监听切歌事件**：用户手动切换歌曲时，保存新索引到 `localStorage`。

```javascript

```

**创建可变的设置副本**。后面的事件监听会修改这个对象，然后保存。

```javascript
    aplayer.value.on('volumechange', () => {
      currentSettings.volume = aplayer.value.volume
      saveAPlayerSettings(currentSettings)
    })
    aplayer.value.on('loopchange', () => {
      currentSettings.loop = aplayer.value.loop
      saveAPlayerSettings(currentSettings)
    })
    aplayer.value.on('orderchange', () => {
      currentSettings.order = aplayer.value.order
      saveAPlayerSettings(currentSettings)
    })
    aplayer.value.on('listfoldchange', () => {
      currentSettings.listFolded = aplayer.value.listFolded
      saveAPlayerSettings(currentSettings)
    })
```

**监听四个设置变化事件**，实时保存到 `localStorage`：

+ `volumechange`：音量变化
+ `loopchange`：循环模式变化
+ `orderchange`：播放顺序变化
+ `listfoldchange`：列表展开/折叠变化

> 注意：这里有个**小 bug**——四个事件共享同一个 `currentSettings` 对象，修改其中一个属性时，其他属性可能已经被之前的操作修改过。但因为每次都完整保存整个对象，所以实际效果是对的。

## 键盘媒体键控制
```javascript
    const handleMediaKeys = (e) => {
      if (!aplayer.value) return
```

**处理键盘事件**，先检查播放器是否存在。

```javascript
      const activeElement = document.activeElement
      const isInput = activeElement.tagName === 'INPUT' 
        || activeElement.tagName === 'TEXTAREA' 
        || activeElement.isContentEditable
```

**检测当前焦点是否在输入框**：

+ `document.activeElement`：当前获得焦点的元素
+ 如果在输入框里，**不拦截空格键**（否则无法输入空格）

```javascript
      if (e.key === 'MediaTrackNext' || e.code === 'MediaTrackNext') {
        e.preventDefault()
        aplayer.value.skipForward()
      } else if (e.key === 'MediaTrackPrevious' || e.code === 'MediaTrackPrevious') {
        e.preventDefault()
        aplayer.value.skipBack()
      } else if (e.key === 'MediaPlayPause' || e.code === 'MediaPlayPause') {
        e.preventDefault()
        aplayer.value.toggle()
      } else if (e.code === 'Space' && !isInput) {
        e.preventDefault()
        aplayer.value.toggle()
      }
    }
```

**支持四种按键**：

| 按键 | 作用 |
| --- | --- |
| `MediaTrackNext` | 下一首（键盘上的多媒体键） |
| `MediaTrackPrevious` | 上一首 |
| `MediaPlayPause` | 播放/暂停 |
| `Space`（空格） | 播放/暂停（不在输入框时） |


`e.preventDefault()`：阻止默认行为（比如空格键滚动页面）。

```javascript

```

**绑定到 window**，全局监听键盘事件。

## Media Session API（系统级媒体控制）
```javascript
    if ('mediaSession' in navigator) {
```

**检测浏览器是否支持 Media Session API**。这是现代浏览器提供的 API，让网页可以**接管系统的媒体控制界面**：

+ 浏览器标签页上的播放按钮
+ 手机锁屏界面的音乐控制
+ 键盘上的多媒体键
+ 蓝牙耳机按键

```javascript
      navigator.mediaSession.setActionHandler('nexttrack', () => {
        aplayer.value?.skipForward()
      })
      navigator.mediaSession.setActionHandler('previoustrack', () => {
        aplayer.value?.skipBack()
      })
      navigator.mediaSession.setActionHandler('play', () => {
        aplayer.value?.play()
      })
      navigator.mediaSession.setActionHandler('pause', () => {
        aplayer.value?.pause()
      })
    }
  }
```

**设置四个动作处理器**：

| 动作 | 对应操作 |
| --- | --- |
| `nexttrack` | 下一首 |
| `previoustrack` | 上一首 |
| `play` | 播放 |
| `pause` | 暂停 |


`?.` 是**可选链操作符**：如果 `aplayer.value` 为 null/undefined，不会报错，直接返回 undefined。

## 启动初始化
```javascript
  preloadAllVideos()
  setTimeout(() => {
    loadAPlayer()
  }, 500)
})
```

**最后执行两个初始化**：

+ `preloadAllVideos()`：立即开始预加载视频
+ `setTimeout(loadAPlayer, 500)`：**延迟 500ms 加载播放器**

延迟的原因可能是：

+ 让页面先渲染完，避免阻塞
+ 让视频先开始加载，减少资源竞争
+ APlayer 的 CSS/JS 较大，延后加载不影响首屏

## onUnmounted —— 组件卸载清理
```javascript
onUnmounted(() => {
  if (aplayer.value) {
    cleanupCrossfade(aplayer.value)
    aplayer.value.destroy()
  }
})
```

**组件销毁时的清理**：

+ `cleanupCrossfade(...)`：清理淡入淡出的定时器/事件监听
+ `aplayer.value.destroy()`：**销毁 APlayer 实例**，释放内存，停止音频播放

> 虽然这个 App.vue 是根组件，理论上不会卸载，但写上清理是良好的编程习惯。如果以后改成路由切换，就不会内存泄漏。
>
